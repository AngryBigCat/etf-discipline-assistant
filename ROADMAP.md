# ETF 投资纪律助手 · 路线图

本文档描述项目愿景、已完成里程碑与后续计划。**以本文档为准维护进度**；编码与模块边界见 [`AGENTS.md`](AGENTS.md)。

## 愿景与方向

本项目是面向个人投资者的 **ETF 投资纪律管理工具**，不是自动交易机器人，也不是荐股系统。

核心目标是帮助用户减少追涨杀跌、满仓冲动、无计划补仓等行为；系统只提供纪律建议，**最终交易由用户人工确认**。

后续演进方向：继续增强「纪律执行、流程自动化、风险提醒、长期复盘」，**不**做成自动交易或收益承诺系统。

## 当前闭环

已形成完整个人投资纪律工具链路：

```text
数据采集 → 指标计算 → 持仓录入 → 仓位管理 → 策略信号 → 交易日志
→ 报告复盘 → AI 复盘 → 回测分析 → 任务中心 → 定时任务 → 邮件通知 → 系统设置
```

## 已实现能力要点

### 数据与看板

- ETF 标的池（`etf_universe` 为主数据源；`config/assets.seed.yaml` 仅初始化）
- 行情采集（东方财富优先，失败切新浪；`PRICE_DATA_SOURCE=mock` 用于测试）
- MA20 / MA60 / MA120 / MA250，回撤、波动率、阶段涨跌幅；数据不足时 `confidence_level=low`

### 持仓、策略与交易

- 手动录入现金与持仓；交易日志**不**自动覆盖持仓快照
- 策略打分（趋势/回撤/反追高/仓位/科创50 特殊规则）与操作建议
- 只观察标的（`enabled_for_signal=false`）不参与买入建议
- 交易纪律检查（如买入金额 ≤ 建议 120% 等阈值）与统计

### 报告、AI 与回测

- 日报/周报模板化生成；AI 复盘默认 mock，可选 OpenAI 兼容 API
- 单标的与组合回测；行情补全与节假日错价清洗；**不代表未来收益**

### 流程自动化

- 任务中心：生成、完成/跳过、安全任务一键执行（白名单）
- 定时任务：工作日收盘后流程、周五周复盘（独立进程，不在 Streamlit 内调度）
- 邮件通知：流程/风险/失败提醒（仅 `.env` 配置，不落库密码）

### 标的 `enabled` 语义


| 配置                                         | 含义                  |
| ------------------------------------------ | ------------------- |
| `enabled=false`                            | 隐藏，不采集，不展示          |
| `enabled=true`, `enabled_for_signal=false` | 可展示/录入，不参与策略信号，仍做风控 |
| `enabled=true`, `enabled_for_signal=true`  | 正常参与仓位计算与策略信号       |


## 规划中阶段

### 阶段 13：外部提醒与通知（部分完成）

**目标**：关键流程与风险事件出现时主动提醒用户查看，而不是自动交易。

**已完成**：邮件提醒（见阶段 13.1）、定时任务失败/成功、高优先级任务、仓位风险、每日流程完成等场景。

**计划中**：

- Telegram / Bark / 企业微信提醒
- 策略信号生成、数据更新失败、AI 复盘完成等专项提醒模板

**边界**：只提醒，不自动交易；不发送「必须买入/卖出」指令；不承诺收益。

### 阶段 14：定时任务增强与运维能力

**目标**：Linux 长期部署更稳定、可观测。

**计划能力**：重试机制、失败原因分类、耗时与成功率统计、节假日/非交易日识别、手动补跑某日流程、systemd 模板完善、日志轮转、数据库备份任务。

**边界**：systemd 只做进程保活；APScheduler 仍为进程内调度；不引入复杂分布式任务系统（除非确有需求）。

### 阶段 15：外部提醒与任务中心联动

**目标**：站内待办 + 外部提醒的统一工作流入口。

**计划能力**：高优先级任务外推、未完成任务每日汇总、定时失败自动生成风险任务、报告/AI 复盘生成后提醒查看、提醒频率控制。

**边界**：外推不含 API Key 明文；提醒不写投资建议；不根据提醒自动改任务状态。

### 阶段 16：高级组合管理

**目标**：更接近真实长期组合管理（仍以提醒与模拟为主）。

**计划能力**：定投日历、再平衡计划、目标仓位偏离趋势、核心/防守/卫星仓分层、QDII 溢价与汇率提示、低相关资产扩展、组合方案对比。

**边界**：再平衡只做提醒和模拟；不自动买卖；不根据回测自动调真实配置。

### 阶段 17：数据质量、备份与恢复

**目标**：提高长期使用可靠性。

**计划能力**：数据质量页、行情缺失与异常价格报告、DB/配置自动备份与恢复、标的池与策略参数变更审计、回测版本管理、一键导出数据包。

**边界**：不自动删历史数据；修复需用户确认；配置恢复前必须备份当前配置。

### 阶段 18：多账户与多组合

**目标**：多账户、多投资计划并行管理。

**计划能力**：多账户与组合配置、账户级快照与组合级目标仓位、分账户纪律统计、组合重复度分析。

**边界**：不接券商实盘；不自动同步真实账户；数据由用户手动录入或导入。

### 阶段 19：导入导出与开放数据接口

**目标**：降低维护成本，提高可迁移性。

**计划能力**：CSV/Excel 导入持仓与交易、导出日志/信号/报告/回测、CLI/API 增强。

**边界**：导入须校验；导入交易日志不自动改持仓快照；导入持仓不自动生成交易记录。

## 长期不做事项

- 不做自动交易
- 不接券商实盘下单接口
- 不做高频交易
- 不做个股荐股
- 不承诺收益
- 不把 AI 作为买卖决策主体
- 不根据回测结果自动修改真实投资配置
- 不自动修改真实持仓、交易日志或策略审核状态
