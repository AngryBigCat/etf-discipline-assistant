# AGENTS.md

本文件用于约束 AI 编码助手在本项目中的行为。

## 项目定位

本项目是 **ETF 投资纪律助手**，不是自动交易系统，不是荐股系统，不是高频量化系统。

系统只提供纪律建议，最终交易由用户人工确认。

## 绝对禁止

AI 编码助手不得实现以下功能，除非用户明确要求并单独确认：

- 自动下单
- 接入券商实盘交易接口
- 高频交易
- 个股荐股
- 自动满仓
- 绕过用户确认的交易执行
- 用大模型直接决定买卖
- 删除用户数据库数据
- 破坏已有迁移兼容
- 擅自修改数据库字段名
- 擅自重构整个项目架构

## 编码原则

- 数据库字段和内部变量使用英文。
- 用户界面必须中文化。
- 页面展示字段通过 `src/ui/labels.py` 模块映射中文。
- 新增数据库字段必须提供兼容迁移（参见 `src/db/schema.py` 的 `apply_schema_migrations`）。
- 不允许为了修 bug 要求用户删库重建。
- 每个阶段只实现本阶段目标，不做过度设计。
- 每次新增核心业务逻辑必须补测试。
- 所有测试必须通过 `pytest`。
- 业务逻辑优先可解释，不追求复杂模型。
- 策略规则必须保留原因说明（`reason` 字段）。

## 模块边界

### 数据层（`src/db/`）

负责：

- 数据库连接
- schema
- repository
- 兼容迁移

不得负责：

- UI 展示
- 策略解释
- 自动交易

### 指标层（`src/indicators/`）

负责：

- 均线
- 回撤
- 波动率
- 阶段涨跌幅

不得负责：

- 买卖决策

### 仓位层（`src/portfolio/`）

负责：

- 账户总资产
- ETF 总市值
- 现金仓位
- 单 ETF 仓位
- 是否超出最大仓位

不得负责：

- 生成交易日志
- 自动修改持仓

### 策略层（`src/strategy/`）

负责：

- 纪律分数
- 操作建议
- 建议金额
- 原因说明

不得负责：

- 自动下单
- 自动修改持仓

### 交易日志层（`src/trading/`）

负责：

- 记录用户实际行为
- 判断是否符合规则
- 统计交易纪律

不得负责：

- 自动下单
- 自动改持仓快照

任何从 `trade_log` 自动推导或覆盖 `holding_snapshot` 的功能，必须作为独立阶段设计，并经过用户明确确认。

### 报告层（`src/reports/`）

负责：

- 聚合账户快照、策略信号、交易日志生成模板化日报/周报
- 格式化报告文本（金额、百分比、标的列表）
- 持久化到 `daily_report` / `weekly_report`

不得负责：

- 自动下单
- 自动改持仓快照
- 用 AI 直接决定买卖（AI 仅可在后续阶段解释/润色报告）

### AI 层（`src/ai_review/`）

只能用于：

- 解释日报与周报
- 复盘交易纪律
- 总结行为模式
- 提醒风险
- 给出纪律层面的改进建议

不得用于：

- 直接决定买卖
- 自动执行交易
- 修改 `trade_log` / `holding_snapshot` / `strategy_signal` / `account_snapshot`
- 预测市场涨跌
- 个股推荐
- 联网新闻分析
- 输出保证收益类表述

必须遵守：

- AI 输出必须包含「不构成投资建议」
- 输出需经 `safety.py` 校验；命中危险词（如「保证收益」「必涨」「必买」「满仓」）时标记 `blocked`
- 不要把 API Key、数据库路径等敏感信息放进 prompt

### 回测层（`src/backtest/`）

负责：

- 读取历史行情模拟定投策略
- 计算回测指标与净值曲线
- 持久化到 `backtest_*` 表

不得负责：

- 自动下单
- 修改 `trade_log` / `holding_snapshot` / `strategy_signal` / `account_snapshot`
- 根据回测结果自动修改 `config.yaml` 或策略参数
- 根据组合回测结果自动调整真实持仓权重或 `config.yaml` 目标仓位
- 预测未来涨跌
- 把回测结果表述为未来收益保证或收益承诺
- 自动寻找最优参数后修改策略配置
- 根据回测结果自动交易

AI 层不得参与回测决策。

### 任务层（`src/tasks/`）

负责：

- 根据行情、持仓、信号、报告、交易日志等状态生成流程任务
- 持久化到 `task_item` 表
- 提供任务完成 / 跳过状态管理
- 对白名单内的安全任务提供一键执行（`execute_task` / `task_action_log`）

不得负责：

- 自动下单或自动修改 `trade_log` / `holding_snapshot` / `account_snapshot` / `strategy_signal`
- 自动审核策略信号或自动调整持仓
- 对非白名单任务提供一键执行；执行按钮必须走白名单
- 根据任务提示自动执行交易
- 把提醒表述为收益承诺或买卖指令
- 邮件 / 手机 / 微信 / 日历等外部通知

## 测试要求

新增或修改以下内容必须补测试：

- 数据库迁移
- 指标计算
- 仓位计算
- 策略打分
- 建议金额
- 交易纪律判断
- 交易日志统计
- 日报/周报生成
- AI 复盘生成与安全校验
- 回测引擎、策略规则、指标计算与持久化
- 回测行情清洗（`load_backtest_prices` / 节假日错价过滤）
- 历史行情补全与多策略对比
- 组合回测（多 ETF 权重、再平衡模拟、`backtest_position` 持久化）
- 任务中心生成规则、状态 upsert 与页面中文化
- 任务快捷执行白名单、执行日志与完成后状态处理
- 任务中心展示分区（今日 / 风险 / 历史）与 badge 中文化
- 配置编辑校验、备份与保存（`src/config/editor.py`）

### 工作流层（`src/workflows/`）

负责：

- 封装行情更新、信号生成、报告生成、AI 复盘等可复用流程
- 供 CLI 脚本与任务中心白名单执行共用

不得负责：

- 自动交易
- 自动修改持仓 / 信号审核状态 / 交易日志

### 配置中心（`src/config/editor.py` / `pages/settings.py`）

负责：

- 读取、校验、保存 `config.yaml`
- 保存前自动备份到 `backups/config/`
- 在页面展示 AI 复盘配置状态（不含 API Key 明文）

不得负责：

- 在配置页面展示 API Key 明文
- 通过配置中心自动交易
- 通过配置中心自动修改真实持仓、交易日志或策略信号
- 根据配置变更自动生成投资建议
- 配置校验失败时保存
- 把参数配置表述为收益承诺

修改配置前必须备份；校验失败不得保存。

## UI 要求

- 用户可见内容必须中文化。
- 不要在页面展示 raw 字段名，例如：
  - `final_score`
  - `action`
  - `suggested_amount`
  - `review_status`
  - `execution_status`
  - `is_rule_based`
- 使用中文：
  - 纪律分数
  - 操作建议
  - 建议金额
  - 审核状态
  - 执行状态
  - 是否符合规则
- 任务中心页面：
  - 使用 `localize_task_status_badge` / `localize_task_priority_badge` 展示状态与优先级
  - 待处理任务只在「今日任务」「风险任务」卡片区展示
  - 已完成 / 已跳过任务只在「历史任务」表格展示
  - 不可执行任务不得显示「执行任务」按钮
- 系统设置页面：
  - 用户可见字段必须中文化，不得展示 raw 配置字段名
  - 不得展示 API Key 明文，仅显示已配置 / 未配置
  - 页面须明确提示：仅维护纪律参数，不构成投资建议，不会自动交易

## 开发流程

每次开发请遵循：

1. 明确本阶段目标
2. 不做阶段外功能
3. 修改代码
4. 增加测试
5. 运行 `pytest`
6. 确认 Streamlit 页面可打开
7. 总结变更和风险点

## 投资风险边界

- 任何页面、报告、提示词中都不得暗示系统可以保证收益。
- 必须保留「系统只用于纪律辅助，不构成投资建议」的定位。
